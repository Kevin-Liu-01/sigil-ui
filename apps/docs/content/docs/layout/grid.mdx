## Import

```tsx
import { SigilGrid, SigilGridCell } from "@sigil-ui/components";
```

## Usage

```tsx
<SigilGrid columns={3}>
  <SigilGridCell>Cell 1</SigilGridCell>
  <SigilGridCell>Cell 2</SigilGridCell>
  <SigilGridCell>Cell 3</SigilGridCell>
  <SigilGridCell>Cell 4</SigilGridCell>
  <SigilGridCell>Cell 5</SigilGridCell>
  <SigilGridCell>Cell 6</SigilGridCell>
</SigilGrid>
```

The grid renders a subtle dot pattern at cell intersections using the `--r-border` color. This is the signature visual of Sigil UI â€” a calibration-grid aesthetic that emerges from the token system.

## Custom gap

```tsx
{/* Pixel value */}
<SigilGrid columns={3} gap={32}>...</SigilGrid>

{/* CSS variable */}
<SigilGrid columns={3} gap="var(--r-rail-gap)">...</SigilGrid>
```

When no `gap` is specified, the grid uses `--r-grid-cell` (default `48px`).

## Cell hover

`SigilGridCell` includes a built-in hover effect that highlights the cell with `--r-surface-elevated`:

```tsx
<SigilGrid columns={3}>
  <SigilGridCell>
    <h3>Feature one</h3>
    <p>Hover to highlight</p>
  </SigilGridCell>
  <SigilGridCell>
    <h3>Feature two</h3>
    <p>Subtle surface shift</p>
  </SigilGridCell>
</SigilGrid>
```

## Without cells

You can use `SigilGrid` alone as a CSS Grid without the cell hover wrapper:

```tsx
<SigilGrid columns={4} gap={16}>
  <div>Plain div</div>
  <div>No hover effect</div>
  <div>Still has the dot grid</div>
  <div>Background pattern</div>
</SigilGrid>
```

## Token integration

| CSS Variable | Used for |
|---|---|
| `--r-grid-cell` | Default gap value |
| `--r-border` | Dot pattern color |
| `--r-surface-elevated` | Cell hover background |
| `--r-cross-arm` | Cross mark arm length (for custom sigil overlays) |
| `--r-cross-stroke` | Cross mark stroke width |
| `--r-rail-gap` | Alternative gap value for rail-aligned layouts |
