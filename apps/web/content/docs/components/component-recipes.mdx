## Import

```tsx
import {
  Modal,
  MultiSelect,
  StatusBadge,
  StatCard,
  DataToolbar,
  AppHeader,
  ChatMessage,
} from "@sigil-ui/components";
```

## Why Recipes

These components are composed from the same token-driven primitives as the rest
of Sigil. They pull from Radix and shadcn-style interaction patterns where that
improves behavior, but every visual decision still flows through `var(--s-*)`.

## CLI

Each recipe is available through the CLI registry:

```bash
npx @sigil-ui/cli add modal multi-select status-badge data-toolbar chat-message
```

The CLI copies the grouped source module and prints any Radix dependencies
needed by that recipe.
